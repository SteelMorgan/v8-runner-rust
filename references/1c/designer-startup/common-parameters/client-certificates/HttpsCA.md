# HttpsCA

## Назначение

Задает источник сертификатов удостоверяющих центров для проверки сертификата сервера.

## Синтаксис

```text
/HttpsCA [-windows] [-linux] [-macos] [-file <path>] [-pwd <password>] [-none]
```

## Связи

- Параметр используется только в тонком клиенте.
- Если ни один из режимов `-windows`, `-linux`, `-macos`, `-file`, `-none` не указан, ключ `/HttpsCA` игнорируется.
- `-windows` использует сертификаты удостоверяющих центров из системного хранилища Windows.
- `-macos` использует сертификаты удостоверяющих центров из системного хранилища macOS.
- `-linux` использует сертификаты удостоверяющих центров из специального каталога Linux.
- `-file <path>` использует сертификаты удостоверяющих центров из указанного файла.
- `-pwd <password>` применим только вместе с `-file <path>`.
- `-none` отключает проверку серверного сертификата корневыми сертификатами.
- `-windows`, `-macos` и `-linux` игнорируются, если указан `-file <path>` или `-none`.

## Примечания

- Если источник сертификатов не задан явно, он определяется в диалоге настройки соединения с информационной базой.
- Для `-file <path>` может потребоваться корректный `-pwd <password>`.
