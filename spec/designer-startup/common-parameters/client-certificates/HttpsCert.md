# HttpsCert

## Назначение

Задает источник клиентского сертификата для HTTPS-соединения.

## Синтаксис

```text
/HttpsCert [-windows] [-linux] [-macos] [-recent] [-auto] [-choose] [-file <path>] [-pwd <password>] [-none]
```

## Связи

- Параметр используется только в тонком клиенте.
- Если ни один из режимов `-windows`, `-linux`, `-macos`, `-file`, `-none` не указан, ключ `/HttpsCert` игнорируется.
- `-windows` использует клиентский сертификат из системного хранилища Windows.
- `-macos` использует клиентский сертификат из системного хранилища macOS.
- `-linux` использует клиентский сертификат из системного хранилища или каталога сертификатов Linux, доступного приложению.
- `-file <path>` использует клиентский сертификат и приватный ключ из указанного файла.
- `-none` запрещает использование клиентского сертификата.
- `-recent`, `-auto` и `-choose` применимы только вместе с `-windows` или `-macos`.
- `-recent` повторно использует ранее выбранный системный сертификат.
- `-auto` автоматически выбирает сертификат из системного хранилища.
- `-choose` всегда открывает выбор сертификата и не использует автоматический выбор.
- `-pwd <password>` применим только вместе с `-file <path>`.
- `-windows` и `-macos` игнорируются, если указан `-file <path>` или `-none`.
- `-choose` игнорируется, если одновременно указан `-auto`.
- При `-none` соединение возможно только с серверами, не требующими клиентский сертификат.

## Примечания

- Если источник сертификата не задан явно, он определяется в диалоге настройки соединения с информационной базой.
- Если в системном хранилище найдено несколько подходящих сертификатов, клиент может открыть системный диалог выбора.
- Если сервер требует клиентский сертификат, для `-file <path>` может понадобиться корректный `-pwd <password>`.
