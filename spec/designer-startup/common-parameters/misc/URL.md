# URL

## Назначение

Передает навигационную ссылку, по которой клиент должен выполнить переход после запуска.

## Синтаксис

```text
/URL <адрес>
```

## Связи

- При поиске цели учитываются разделители, переданные через `/Z`.
- Если подходящее клиентское приложение уже запущено, платформа пытается использовать его.
- Для внешней ссылки формата `e1c:` платформа ищет запущенный клиент с той же строкой соединения.
- Если исходная навигационная ссылка содержит только адрес информационной базы без локальной ссылки, переход не выполняется, а активируется основное окно найденного клиента.
- Если подходящий клиент не найден, строка соединения берется из параметра `/URL` и приложение запускается обычным способом.
- Если указана локальная ссылка, после запуска выполняется переход по ней.
- Для ссылок `http` и `https` всегда запускается или активируется тонкий клиент.

## Примечания

- В найденном клиентском приложении не должно быть открыто модальное или блокирующее окно.
- Если переход выполнить не удалось, клиент продолжает работу.
